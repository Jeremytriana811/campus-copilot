# Microsoft Story Notes

## Why this project is Microsoft-style
- It is a copilot workflow grounded in organizational documents
- It exposes evidence, citations, and refusal behavior
- It includes admin-managed ingestion and evaluation
- It separates probabilistic AI from deterministic planning and scheduling
- It is local-first and cloud-ready by design

## Key design choices
- Student Copilot + Admin Center + Evaluation Center
- Retrieval is modular
- Planning logic is auditable
- Scheduling uses constraint solving
